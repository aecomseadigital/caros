# Handover — Cross-Tool AI Benchmark

> Audience: an incoming coding agent (e.g. Codex) with **no prior context**. This doc
> is self-contained — you do not need the original chat. Read §0–§4, then §7 (tools)
> and §11 (what's left to do).

---

## 0. TL;DR

Build/operate a harness that compares three AI assistants on **accuracy, response
time, and cost**, and emit an **executive decision report**. The harness scaffold is
already written and syntax-checked. Your job is to finish the live-integration parts
(web-UI selectors, auth, datasets, a real run) and produce the report.

- Code lives in `caros/evals/cross-tool/` (this folder).
- Offline smoke test works today: `just smoke` (Caros only, committed sample data).
- Everything blocked on a human is listed in §11 and §13.

---

## 1. Mission & deliverable

Compare, for the company's decision-makers:

| Target | What it is | Access |
|---|---|---|
| **Oscar** | Internal corporate web chatbot (RAG over internal corpus) | Web UI only, no free API |
| **Microsoft Copilot Chat (Basic)** | Free M365 Copilot Chat (`m365.cloud.microsoft/chat`) | Web UI only, no API at Basic tier |
| **Caros** | This repo's agentic CLI (goose-based; tool-calling/MCP) | Scriptable CLI (`caros run`) |

**Deliverable:** `output/executive-report.md` — per-track accuracy, latency (median ±
IQR), and cost, plus a recommendation. Generated by `report/aggregate.mjs`.

---

## 2. Key facts about Caros (the third target)

- goose fork. Invoked via the `caros` CLI; fully scriptable and headless.
- Backs onto an **Azure OpenAI gateway**; routes **server-side** between
  `gpt-5.4-mini` and `gpt-5.4-nano` — you **cannot** force a model per request. Record
  which served each request via the session log / `x-caros-deployment` if exposed.
- Auth is Microsoft Entra (device-code on CLI). Run `caros login` once.
- Has two **existing** eval harnesses you should reuse for the agentic track instead of
  re-implementing: `../open-model-gym/` and `../harbor/` (see §7.7–7.8).

---

## 3. Locked decisions (do not relitigate)

1. Third tool is **Caros** (not a model-vs-model study — it's tool-vs-tool).
2. Cover **all four tracks**: Factual Q&A, Summarization & RAG, Reasoning, Coding & agentic.
3. **Automate both** web tools with Playwright (the human accepted Copilot's ToS risk —
   but the mitigations in §13 are mandatory and enforced in code).
4. Output is an **executive decision report** → curated subset (~100–150 items/track),
   not exhaustive full-dataset runs.
5. Judge model is a **different family from the targets** (targets are GPT/AOAI →
   judge with **Claude**) to reduce self-preference bias.

---

## 4. The fairness rule (most important design constraint)

The three tools are architecturally different (RAG chat vs web-grounded chat vs agent).
Therefore:

- **Report per-track. Never blend tracks into one score.** `report/aggregate.mjs`
  enforces this; do not "average to a single number."
- The **Coding & agentic** track is a **Caros-differentiation** track: the web chat
  tools can't execute code or use tools. Label it as such; don't present it as
  apples-to-apples.
- For the **RAG** track, paste identical source passages **into the prompt** for all
  three tools (tests reading/synthesis, not each tool's private corpus). This is both
  fairer and required by the data policy (§10/§13).

## 5. Scenario tracks → suitable suites

| Track (`vars.track`) | Suite(s) | Grading |
|---|---|---|
| `qa` | SimpleQA, IFEval | SimpleQA → LLM judge; IFEval → official verifier (not promptfoo asserts) |
| `rag` | HotpotQA-distractor (context in-prompt), optional FRAMES | LLM judge (faithfulness + correctness) |
| `reasoning` | GPQA-Diamond, MMLU-Pro subset | objective exact letter match (no judge) |
| `coding` | SWE-bench Verified subset, GAIA L1 | Caros: execution via open-model-gym/harbor; web tools: judged code text |

Explicitly **excluded** as overkill/irrelevant: GSM8K (saturated), WebArena/OSWorld
(web-action agents), full AgentBench, τ-bench (needs simulated-user + tool APIs the web
tools don't expose) — optional stretch only.

## 7. Tools & commands reference

> First thing to do on a new machine: confirm CLI flag names against the installed
> versions (`npx promptfoo eval --help`, `caros run --help`, `just --list`). Where this
> doc is unsure, it says so — verify rather than assume.

### 7.1 This harness — Justfile recipes
Run from `caros/evals/cross-tool/`. (`just --list` to see all.)

| Command | Does |
|---|---|
| `just install` | `npm install` + `npm run browsers` (Chromium) |
| `just fetch 120` | Pull+subset datasets, ~120 rows/suite → `datasets/<track>/` |
| `just login oscar` / `just login copilot` | Interactive sign-in, saves `.auth/<t>.json` |
| `just smoke` | **Offline**: Caros only, sample data, deterministic asserts (no judge key) |
| `just track qa 5` | One track vs all consented targets, 5 reps (for latency) |
| `just eval` | Full run (config default) |
| `just view` | promptfoo side-by-side viewer |
| `just report` | Build `output/executive-report.md` |

### 7.2 This harness — npm scripts
| Script | Command |
|---|---|
| `npm run eval` | `promptfoo eval -c promptfooconfig.yaml` |
| `npm run view` | `promptfoo view` |
| `npm run report` | `node report/aggregate.mjs` |
| `npm run browsers` | `playwright install chromium` |

### 7.3 Dataset fetcher — `datasets/fetch_datasets.py`
Needs `pip install datasets`. Auto-generates promptfoo test rows with the right
grading per suite (objective for MC, `llm-rubric` for open-ended).
```bash
python datasets/fetch_datasets.py --n 120          # all suites, 120 rows each
python datasets/fetch_datasets.py --suite gpqa --n 50
```
- Suites implemented: `simpleqa` (qa), `gpqa` (reasoning), `mmlu_pro` (reasoning),
  `hotpotqa` (rag). HF ids are at the top of the file — **verify against the Hub**;
  GPQA is **gated** (run `huggingface-cli login` and accept terms).
- Stubbed with notes (need special handling): **IFEval** (official verifier),
  **GAIA** (gated, agentic), **SWE-bench Verified** (container execution).

### 7.4 Web auth + Playwright
- One-time login (saves storage state): `node providers/lib/browser.js --login oscar`
- **Discover selectors** for a web UI (the main TODO):
  `npx playwright codegen https://oscar.aecom.global/home` — click into the input,
  the send button, an assistant message bubble, and the "generating" indicator; copy
  those selectors into `promptfooconfig.yaml` under each provider's `selectors:`.
- `HEADLESS=1` env to run headless once selectors are stable.

### 7.5 promptfoo CLI (orchestrator)
Confident flags: `-c/--config`, `--filter-providers <regex>`, `--repeat <n>`,
`--max-concurrency <n>`, `--no-cache`, `--output <path>`, `--no-table`.
```bash
npx promptfoo eval -c promptfooconfig.yaml --filter-providers caros --repeat 5
npx promptfoo view                 # browser UI
```
- **Track selection:** the config's `tests:` key points at a dataset file. The Justfile
  passes `--tests <path>` — **verify this flag exists in your promptfoo version**; if
  not, either (a) edit `tests:` in the config per run, or (b) parameterize it with env
  substitution: `tests: ${TESTS_FILE}` and set `TESTS_FILE=datasets/qa/qa.jsonl`.
  promptfoo auto-loads `.env` and substitutes `${VAR}` in the config.
- The judge is set once in `defaultTest.options.provider` (Claude). `llm-rubric` rows
  route to it; objective asserts (`javascript`/`icontains`/`regex`) need no judge.

### 7.6 Caros CLI (the target under test)
Run `caros --help` / `caros run --help` for the authoritative surface. Confirmed:
```bash
caros login                              # one-time Entra device-code sign-in
caros run -i -                           # prompt on STDIN (what caros.js uses)
caros run -t "your prompt"               # inline text
caros run --recipe path/to.yaml --params k=v   # parameterized agentic task
caros run --system "extra system prompt"
caros run -i - --no-session              # no persisted session
caros session list --format json         # inspect sessions (tokens/turns)
```
- `caros.js` shells out with `run -i - --no-session`, measures TTFT from first stdout
  byte, and best-effort parses the newest session jsonl under
  `~/.config/caros/sessions/` for token usage + deployment.
- For agentic rows, set `vars.recipe` and it switches to `--recipe`.

### 7.7 Reused harness — `../open-model-gym/` (agentic/coding track for Caros)

## 10. Grading model & guardrails (enforced in code)

- **Objective first:** multiple-choice (GPQA/MMLU) and format constraints grade with
  `javascript`/`icontains`/`regex` asserts — no judge, no bias.
- **LLM judge** (`llm-rubric`, Claude) only for open-ended QA/RAG. Rubrics in
  `judge/rubrics/` bake in: don't reward length, grade vs reference, swap A/B order for
  pairwise, human spot-check 10–20%.
- **Guardrail 1 — data policy:** `providers/lib/policy.js#assertWebDataClass` throws if a
  row isn't `data_class: public|synthetic`. Web tools (esp. Copilot Basic) may lack
  Enterprise Data Protection — **never send real confidential data**.
- **Guardrail 2 — Copilot gate:** `assertCopilotConsent` throws unless `ALLOW_COPILOT=1`
  **and** `.auth/copilot.consent` exists (create only after sign-off).

## 13. Risks & compliance (read before any web run)

- **Copilot ToS / account suspension (accepted, mitigate):** dedicated non-privileged
  test account; human-like pacing (providers sleep 2.5–4 s); `maxConcurrency: 1`; off-hours;
  audit log; **manager/IT sign-off first**; **stop if blocked**, never evade. The code
  refuses to run Copilot until `ALLOW_COPILOT=1` + `.auth/copilot.consent`.
- **Data privacy (biggest substantive risk):** Copilot Basic may lack Enterprise/
  Commercial Data Protection → **public/synthetic eval data only** (enforced). This also
  fixes the RAG-corpus asymmetry (context is provided in-prompt).
- **Opaque/shifting models:** Oscar's and Copilot's underlying models are undisclosed and
  may change; Caros routes mini/nano server-side. Record what you can; caveat the rest;
  results are a **dated snapshot**.
- **Browser fragility:** UI drift, SSO timeouts, rate limits → accessibility-tree (not
  pixel) selectors, persisted auth, retries, manual fallback per item.
